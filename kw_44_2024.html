Github

GitHub ist eine Plattform für das Versionsmanagement und die Zusammenarbeit an Softwareprojekten. Sie basiert auf dem Versionskontrollsystem Git und ermöglicht es Entwicklern, Änderungen am Code zu verfolgen, ältere Versionen wiederherzustellen und gemeinsam an Projekten zu arbeiten. GitHub bietet einen zentralen, Cloud-basierten Speicherort für Projekte und ist besonders nützlich für Teams oder die Arbeit an Open-Source-Projekten. Um Dateien auf GitHub hochzuladen, beginnt man normalerweise mit dem Terminal, entweder direkt in einem Editor wie Visual Studio Code oder über das normale Terminal. Der erste Schritt besteht darin, die gewünschten Dateien (zb. Der Wochenbericht) zur Staging Area hinzuzufügen. Dies geschieht mit dem Befehl git add . Der Punkt (.) steht dabei für alle Dateien im aktuellen Verzeichnis. Auf diese Weise werden alle Änderungen für den späteren Upload vorgemerkt. Nach dem Hinzufügen zur Staging Area folgt das sogenannte „Commit“, um die Änderungen lokal zu speichern. Der Befehl hierfür lautet git commit -m "Nachricht". Die Option -m erlaubt es, eine kurze Nachricht zu den Änderungen hinzuzufügen, z.B. git commit -m "Upload README.md". Diese Nachricht erklärt kurz, was in diesem Commit geändert wurde und macht die Versionsgeschichte übersichtlicher. Durch Commits wird eine nachvollziehbare Historie des Projekts erstellt, die es ermöglicht, bestimmte Änderungen zu identifizieren und bei Bedarf zu früheren Versionen zurückzukehren. Der letzte Schritt, um die Dateien auf GitHub verfügbar zu machen, ist der Befehl git push. Dieser Befehl überträgt die lokalen Commits vom Computer in das Remote-Repository auf GitHub. Dadurch wird der aktuelle Stand des Projekts auf GitHub aktualisiert und anderen Teammitgliedern oder interessierten Benutzern zugänglich gemacht. Durch das Hochladen auf GitHub ist der Code nicht nur sicher gespeichert, sondern auch von anderen Geräten aus zugänglich und kann von den Beitragenden bearbeitet werden. GitHub bietet neben der Versionskontrolle viele weitere Funktionen. Zum Beispiel können über Issues Fehler gemeldet oder neue Funktionen vorgeschlagen werden. Ausserdem gibt es Pull Requests, mit denen Änderungen überprüft und in den Hauptcode integriert werden können. Auch Dokumentationen können direkt in den Repositories abgelegt werden. Damit ist GitHub nicht nur ein reiner Speicherort, sondern eine komplette Plattform für Softwareentwicklung und Zusammenarbeit.
